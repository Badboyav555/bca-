# SYSTEM ANALYSIS AND DESIGN (SAD)

# UNIT – INPUT/OUTPUT AND FORM DESIGN

---

# Introduction to Input and Output Design

In every information system:
```text
Input and output are very important parts
```

A system receives:
- data as input
- information as output

If the input is incorrect:
```text
The output will also be incorrect
```

Therefore:
- proper input design
- proper output design

are necessary for an efficient system.

---

# What is Input Design?

Input Design is:
```text
The process of designing methods and forms used for entering data into a system
```

The main goal of input design is:
- easy data entry
- accurate data entry
- reducing errors
- improving efficiency

---

# Real Life Example

ATM Machine:
- entering PIN
- entering withdrawal amount

are examples of input design.

If the keypad or screen is confusing:
```text
Users may enter wrong information
```

---

# Objectives of Input Design

1. Reduce input errors
2. Improve data accuracy
3. Speed up data entry
4. Make system user friendly
5. Improve system efficiency

---

# Importance of Input Design

Good input design:
- improves data quality
- reduces mistakes
- saves time
- increases efficiency

Poor input design:
- creates confusion
- increases errors
- wastes time

---

# Input Design Methodologies

Different methodologies are used while designing input forms.

---

# 1. Simple Design Methodology

Input forms should be:
```text
Simple and easy to understand
```

---

# Example

Registration forms should avoid:
- unnecessary fields
- complicated instructions

---

# 2. Consistency Methodology

All forms should maintain:
- same layout
- same style
- same button placement

---

# Example

If submit button is always placed at the bottom right:
```text
Users can use forms more easily
```

---

# 3. Validation Methodology

The system should:
```text
Reject invalid or incorrect input
```

---

# Example

Phone number field should allow:
```text
Only numeric values
```

---

# 4. User Friendly Methodology

Forms should use:
- simple language
- clear instructions
- readable labels

---

# Example

Using:
```text
Name
Email
Phone Number
```

instead of technical terms.

---

# Characteristics of Good Input Design

1. Simple
2. Accurate
3. User friendly
4. Fast
5. Error free

---

# Difference Between User Manual and Operator Manual

| User Manual | Operator Manual |
|---|---|
| Prepared for end users | Prepared for system operators |
| Explains basic usage | Explains technical operations |
| Uses simple language | Uses technical language |
