# SYSTEM ANALYSIS AND DESIGN (SAD)

# UNIT – FEASIBILITY STUDY, COST BENEFIT ANALYSIS AND SYSTEM PROPOSAL

---

# Introduction to Feasibility Study

Kisi bhi software ya system ko develop karne se pehle:
```text
Yeh check karna zaruri hota hai ki project possible hai ya nahi
```

Isi checking process ko:
```text
Feasibility Study
```
kehte hain.

Feasibility study help karti hai decide karne mein:
- project practical hai ya nahi
- project profitable hai ya nahi
- project successful ho sakta hai ya nahi

---

# Real Life Example

Suppose:
ek school:
```text
Online Attendance System
```
banana chahta hai.

Development start karne se pehle management check karegi:
- computers available hain?
- internet available hai?
- teachers system use kar paayenge?
- budget available hai?

Ye complete checking process:
```text
Feasibility Study
```
kehlati hai.

---

# Definition of Feasibility Study

Feasibility Study:
```text
Ek process hai jisme proposed system ko evaluate kiya jata hai ki project technically, operationally aur economically possible hai ya nahi
```

---

# Objectives of Feasibility Study

1. Project possibility check karna
2. Risk kam karna
3. Time aur money save karna
4. Problems ko pehle identify karna
5. Management ko decision lene mein help karna

---

# Importance of Feasibility Study

Agar feasibility study na ho:
- project fail ho sakta hai
- budget increase ho sakta hai
- users system reject kar sakte hain
- software properly work nahi karega

Feasibility study ensure karti hai:
```text
Better planning aur successful system development
```

---

# Types of Feasibility

Main types:
1. Technical Feasibility
2. Operational Feasibility
3. Economic Feasibility

---

# 1. Technical Feasibility

# Introduction

Technical feasibility check karti hai:
```text
Required technology, hardware aur software available hain ya nahi
```

Ye determine karti hai:
- system technically possible hai?
- required devices available hain?
- staff technically skilled hai?

---

# Real Life Example

Suppose:
hospital:
```text
AI based patient monitoring system
```
banana chahta hai.

Lekin:
- computers outdated hain
- internet slow hai
- staff ko technical knowledge nahi hai

Toh project:
```text
Technically feasible nahi hoga
```

---

# Questions Asked in Technical Feasibility

1. Required hardware available hai?
2. Required software available hai?
3. Technology reliable hai?
4. Technical support available hai?
5. Staff system handle kar sakta hai?

---

# Advantages of Technical Feasibility

1. Technical failure reduce hota hai
2. Technology related problems pehle identify ho jati hain
3. System performance improve hota hai

---

# 2. Operational Feasibility

# Introduction

Operational feasibility check karti hai:
```text
Users new system ko accept aur operate kar paayenge ya nahi
```

Ye study karti hai:
- user acceptance
- employee training
- system usability

---

# Real Life Example

Suppose:
company:
```text
Advanced Payroll Software
```
install karti hai.

Lekin employees:
- computer properly use nahi kar paate

Toh operational problems aayengi.

---

# Questions Asked in Operational Feasibility

1. Users system accept karenge?
2. Training required hai?
3. System easy to use hai?
4. Work efficiency improve hogi?

---

# Importance of Operational Feasibility

Chahe technology best ho:
```text
Agar users system use nahi karenge toh project fail ho jayega
```

---

# Advantages

1. User satisfaction improve hoti hai
2. Employee resistance kam hota hai
3. Smooth working ensure hoti hai

---

# 3. Economic Feasibility

# Introduction

Economic feasibility check karti hai:
```text
Project financially beneficial hai ya nahi
```

Ye compare karti hai:
- project cost
- expected benefits

---

# Real Life Example

Suppose:
```text
Project Cost = ₹10 lakh
```

Expected Benefit:
```text
₹50 lakh
```

Toh project economically feasible hai.

Lekin agar:
```text
Benefit sirf ₹1 lakh ho
```

Toh project useful nahi hoga.

---

# Costs Included in Economic Feasibility

1. Hardware Cost
2. Software Cost
3. Employee Salary
4. Training Cost
5. Maintenance Cost

---

# Benefits Included

1. Increased profit
2. Faster work
3. Reduced errors
4. Better customer service

---

# Advantages of Economic Feasibility

1. Unnecessary expenses control hote hain
2. Better investment decisions milte hain
3. Project profitability ensure hoti hai

# Difference Between Feasibility Study and Cost Benefit Analysis

| Feasibility Study | Cost Benefit Analysis |
|---|---|
| Project possibility check karta hai | Cost aur benefits compare karta hai |
| Technical factors include karta hai | Mainly financial comparison karta hai |
| Broad concept hai | Feasibility study ka part hai |

---

# Important Points to Remember

- Feasibility study project possibility check karti hai.
- Technical feasibility technology availability check karti hai.
- Operational feasibility user acceptance check karti hai.
- Economic feasibility financial benefits check karti hai.
- Constraints system limitations hoti hain.
- Objectives system goals define karte hain.
- Feasibility report project feasibility explain karti hai.
- Cost benefit analysis expenses aur benefits compare karta hai.
- System proposal complete proposed system explain karta hai.
