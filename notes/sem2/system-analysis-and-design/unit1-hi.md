# Introduction to System Analysis and Design

Aaj ke modern world mein:
- banks
- schools
- hospitals
- railway systems
- online shopping websites

sab:
```text
Computerized systems
```
use karte hain.

Kisi bhi software ya information system ko properly develop karne ke liye:
```text
Planning aur proper process
```
ki zarurat hoti hai.

Isi process ko:
```text
System Development Life Cycle (SDLC)
```
kehte hain.

---

System:


karne ke liye milkar kaam karte hain.

# SYSTEM DEVELOPMENT LIFE CYCLE (SDLC)

# Introduction

SDLC:
```text
Software/System develop karne ki step-by-step process
```
hai.

Ye:
- planning
- designing
- development
- testing
- maintenance

sab phases ko include karti hai.

---

# Why SDLC Important

Without planning:
- software fail ho sakta hai
- cost increase ho sakti hai
- errors jyada ho sakte hain

SDLC:
```text
Proper management aur successful development
```
ensure karta hai.

---

# Real Life Example

Suppose:
ghar banana hai.

Direct construction start nahi karte:
1. Planning
2. Budget
3. Design
4. Material selection
5. Construction

sab phases follow karte hain.

Software development bhi:
```text
Isi tarah hota hai
```

---

# Phases of SDLC

Main phases:
1. Preliminary Investigation
2. Feasibility Study
3. System Analysis
4. System Design
5. Coding/Development
6. Testing
7. Implementation
8. Maintenance

# 2. Feasibility Study

# Introduction

Feasibility Study:
```text

```

---

# Types of Feasibility

1. Technical Feasibility
2. Economic Feasibility
3. Operational Feasibility
4. Schedule Feasibility

---

# Technical Feasibility

Check:
- hardware available?
- software available?
- technology possible?

---

# Example

Suppose:

```text

```
banana chahta hai.

- computers weak hain

Toh project technically feasible nahi ho sakta.

---

# Economic Feasibility

Check:
```text
Project cost beneficial hai ya nahi
```

---

# Example

Suppose:

```text
10 lakh
```

```text

```

---

# Operational Feasibility

Check:
```text

```

---

# Example

Suppose:

```text
Computer knowledge nahi rakhte
```

Toh operational problem hogi.

---

# Schedule Feasibility

Check:
```text
Project time par complete hoga ya nahi
```

# Difference Between Analysis and Design

| Analysis | Design |
|---|---|
| Problem study | Solution planning |
| Requirements collect | Structure create |
| Existing system focus | New system focus |

---

# Important Points to Remember

- SDLC software development ki step-by-step process hai.
- Preliminary investigation first phase hai.
- Feasibility study project possibility check karti hai.
- Analysis requirements collect karta hai.
- Design blueprint create karta hai.
- Coding actual development hai.
- Testing errors remove karti hai.
- Maintenance updates provide karti hai.
- Fact finding information collect karne ki process hai.
