# SYSTEM TESTING AND QUALITY ASSURANCE

# Introduction to System Testing

System Testing is:
```text
A process of checking whether the complete system works correctly or not
```

The main purpose of testing is:
- finding errors
- removing bugs
- improving performance
- ensuring reliability

---

# Real Life Example

Suppose:
an Online Shopping Website has been developed.

Testing will check:
- whether login works properly
- whether payment is successful
- whether orders are placed correctly
- whether the website crashes or not

If any problem is found:
```text
It is fixed before final use
```

---

# Objectives of System Testing

1. Detect errors and bugs
2. Verify correct output
3. Improve system performance
4. Check security
5. Ensure system reliability

---

# Importance of System Testing

Without testing:
- software may crash
- incorrect results may occur
- users may reject the system

Good testing:
```text
Improves software quality and reliability
```

---

# Types of System Testing

1. Unit Testing
2. Integration Testing
3. System Testing
4. Acceptance Testing

---

# 1. Unit Testing

In Unit Testing:
```text
Small modules or components are tested individually
```

---

# Example

In a calculator application:
- addition function
- subtraction function

are tested separately.

---

# Advantages

1. Early bug detection
2. Easy debugging
3. Better code quality

---

# 2. Integration Testing

Integration Testing:
```text
Tests combined modules together
```

---

# Example

In Online Shopping System:
- login module
- payment module
- order module

are checked together.

---

# 3. System Testing

System Testing:
```text
Tests the complete software system
```

---

# Example

In Banking Software:
- login
- money transfer
- balance checking

all features are tested.

---

# 4. Acceptance Testing

Acceptance Testing:
```text
Is performed by users or clients
```

It checks whether:
```text
System meets user requirements or not
```

---

# Example

School staff and principal test School Management Software before final approval.

---

# QUALITY ASSURANCE (QA)

# Introduction

Quality Assurance is:
```text
A process that ensures software quality during development
```

QA focuses on:
- preventing errors
- improving processes
- maintaining quality standards

---

# Difference Between Testing and Quality Assurance

| Testing | Quality Assurance |
|---|---|
| Detects errors | Prevents errors |
| Product oriented | Process oriented |
| Performed after development stages | Performed throughout development |

---

# Objectives of Quality Assurance

1. Deliver high quality software
2. Reduce errors
3. Improve user satisfaction
4. Maintain standards

---

# Quality Assurance Activities

1. Code review
2. Documentation checking
3. Performance testing
4. Process monitoring

---

# Real Life Example

Before launching a mobile app:
- speed testing
- security testing
- interface checking

are part of QA activities.

---

# SYSTEM IMPLEMENTATION

# Introduction

System Implementation is:
```text
The process of installing and using the developed system in a real environment
```

It is an important phase of SDLC.

---

# Steps in System Implementation

1. Planning
2. Installation
3. Training
4. Data Conversion
5. System Testing
6. Going Live

---

# 1. Planning

Resources:
- time
- budget
- staff

are planned before implementation.

---

# 2. Installation

Required software and hardware are installed.

---

# Example

Installing payroll software in office computers.

---

# 3. Training

Users are trained:
```text
To use the new system properly
```

---

# Example

Employees learning attendance software.

---

# 4. Data Conversion

Old system data is transferred:
```text
Into the new system
```

---

# Example

Old student records transferred into new database.

---

# 5. System Testing

After installation:
```text
The system is tested again
```

---

# 6. Going Live

The system officially starts working for users.

# THREAT AND RISK ANALYSIS

# Introduction

Threat:
```text
A possible danger to the system
```

Risk:
```text
Possible damage caused by a threat
```

Threat and Risk Analysis:
```text
The process of identifying dangers and analyzing their impact
```

---

# Types of Threats

1. Virus
2. Hacking
3. Data theft
4. Hardware failure
5. Natural disasters

---

# Example

A hacker trying to access bank database illegally.

---

# Risk Analysis Process

1. Identify threats
2. Measure risks
3. Decide protection methods
4. Create security plan

---

# Importance of Risk Analysis

1. Improves security
2. Reduces financial loss
3. Improves system reliability

---

# Difference Between Threat and Risk

| Threat | Risk |
|---|---|
| Source of danger | Possible damage caused by danger |
| Example: Virus | Example: Data loss |

---

- Software maintenance improves and updates software.
- Data security protects important data.
- Disaster recovery restores system after failure.
- Ethics maintain professional behavior.

- Risk means possible loss or damage.
