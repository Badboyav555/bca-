# Introduction to System Analysis and Design

In today’s modern world:
- banks
- schools
- hospitals
- railway systems
- shopping websites

all use:
```text
Computerized Systems
```

To develop any software or information system properly:
```text
Planning and step-by-step process
```
is required.

This complete process is called:
```text
System Development Life Cycle (SDLC)
```

---

# What is a System?

A system is:
```text
A group of connected components working together to achieve a goal
```

---

# Real Life Example of a System

# School System

A school contains:
- students
- teachers
- classrooms
- exams
- fees

All these work together to form:
```text
School System
```

---

# What is an Information System?

An Information System is:
```text
A computer-based system used to collect, process, store and manage data
```

---

# Example

ATM Machine:
- takes user input
- verifies account
- gives cash
- updates records

This is an:
```text
Information System
```

---

# SYSTEM DEVELOPMENT LIFE CYCLE (SDLC)

# Introduction

SDLC is:
```text
A step-by-step process used to develop software or systems
```

It includes:
- planning
- designing
- coding
- testing
- maintenance

---

# Why SDLC is Important

Without proper planning:
- software may fail
- cost may increase
- errors may occur

SDLC helps in:
```text
Proper management and successful development
```

---

# Real Life Example

Suppose you want to build a house.

You do not directly start construction.

First:
1. Planning
2. Budget
3. Design
4. Material collection
5. Construction

are done.

Software development also follows:
```text
A similar process
```

---

# Phases of SDLC

Main phases are:
1. Preliminary Investigation
2. Feasibility Study
3. System Analysis
4. System Design
5. Coding / Development
6. Testing
7. Implementation
8. Maintenance

# 1. Preliminary Investigation

# Introduction

This is:
```text
The first phase of SDLC
```

In this phase:
- problems are identified
- project need is analyzed

---

# Example

Suppose a college uses:
```text
Manual attendance system
```

Problems:
- time waste
- mistakes in records
- difficult calculations

Management decides to create:
```text
Computerized attendance system
```

This is an example of:
```text
Preliminary Investigation
```

---

# Objectives of Preliminary Investigation

1. Identify problems
2. Understand user needs
3. Study existing system

---

# 2. Feasibility Study

# Introduction

Feasibility Study checks:
```text
Whether the project is possible or not
```

---

# Types of Feasibility

1. Technical Feasibility
2. Economic Feasibility
3. Operational Feasibility
4. Schedule Feasibility

---

# Technical Feasibility

Checks:
- required hardware available?
- required software available?
- technology possible?

---

# Example

Suppose a school wants:
```text
AI-based software
```

But computers are old and weak.

Then the project may not be:
```text
Technically feasible
```

---

# Economic Feasibility

Checks:
```text
Whether project cost is beneficial or not
```

---

# Example

Project Cost:
```text
10 lakh rupees
```

Benefit:
```text
Only 1 lakh
```

Then project may not be useful economically.

---

# Operational Feasibility

Checks:
```text
Whether users can operate the system properly
```

---

# Example

If employees do not know computer usage:
```text
Operational problems may occur
```

---

# Schedule Feasibility

Checks:
```text
Whether project can be completed on time
```

---

# INITIAL INVESTIGATION

# Introduction

Initial Investigation is:
```text
First practical step of SDLC
```

It helps understand:
- organizational problems

- project scope

---

# Objectives of Initial Investigation

1. Identify problem
2. Study current system
3. Collect user requirements
4. Find possible solutions

---

# Real Life Example

Suppose library uses:
```text
Manual book issue system
```

Problems:
- lost records
- slow searching
- fine calculation mistakes

Investigation identifies these issues.

---

# Determining User Requirements

# Introduction

User Requirements define:
```text
What users expect from the system
```

---

# Types of Requirements

1. Functional Requirements
2. Non-Functional Requirements

---

# Functional Requirements

Defines:
```text
What system should do
```

---

# Example

ATM:
- cash withdrawal
- balance inquiry
- mini statement

---

# Non-Functional Requirements

Defines:
```text
How system should perform
```

---

# Example

- speed
- security
- reliability

---

# Importance of User Requirements

If requirements are not collected properly:
```text
Software may fail
```

---

# Example

Suppose school software needed:
```text
Online fee payment feature
```

But developer forgot to add it.

Result:
```text
Users become unhappy
```

---

# Fact Finding Process

# Introduction

Collecting information about system:
```text
Fact Finding
```
is called.

---

# Objectives

1. Understand existing system
2. Identify problems
3. Collect requirements

---

# Fact Finding Techniques

1. Interview
2. Questionnaire
3. Observation
4. Document Review
5. Sampling

---

# 1. Interview Technique

# Introduction

Asking questions directly from users.

---

# Example

Bank manager interview:
- daily problems
- required features

---

# Advantages

1. Detailed information
2. Direct communication
3. Better understanding

---

# Disadvantages

1. Time consuming
2. Expensive

---

# 2. Questionnaire Technique

# Introduction

Written questions given to users.

---

# Example

Google Form survey.

---

# Advantages

1. Fast
2. Cheap
3. Large audience coverage

---

# Disadvantages

1. Incomplete answers
2. Less interaction

---

# 3. Observation Technique

# Introduction

Watching users while they work.

---

# Example

Observing cashier using billing system.

---

# Advantages

1. Real information
2. Accurate understanding

---

# Disadvantages

1. Time consuming
2. Users may feel uncomfortable

---

# 4. Document Review

# Introduction

Studying existing:
- reports
- files
- records

---

# Example

- attendance registers
- invoices
- customer records

---

# 5. Sampling

# Introduction

Studying:
```text
Small part of large data
```

---

# Example

Checking 500 records from 10000 customer files.

---

# Difference Between Analysis and Design

| Analysis | Design |
|---|---|
| Studies problem | Creates solution |
| Collects requirements | Creates structure |
| Focus on existing system | Focus on new system |

---

# Important Points to Remember

- SDLC is step-by-step software development process.
- Preliminary investigation is first phase.
- Feasibility study checks project possibility.
- Analysis collects requirements.
- Design creates blueprint.
- Coding develops actual software.
- Testing removes errors.
- Maintenance provides updates.
- Fact finding collects information.

---

# Conclusion

System Development Life Cycle:
```text
Foundation of successful software development
```

Proper:
- planning
- analysis
- investigation
- requirement gathering

help create better systems.

Fact finding techniques like:
- interviews
- observation
- questionnaires

help developers understand user needs and create useful software systems.
