# SYSTEM TESTING AND QUALITY ASSURANCE

# Introduction to System Testing

System Testing:
```text
Ek process hai jisme complete system ko check kiya jata hai ki wo properly work kar raha hai ya nahi
```

Testing ka main purpose hota hai:
- errors detect karna
- bugs remove karna
- system performance improve karna
- system reliability ensure karna

---

# Real Life Example

Suppose:
ek Online Shopping Website banayi gayi hai.

Testing mein check kiya jayega:
- login properly work kar raha hai ya nahi
- payment successful ho raha hai ya nahi
- order place ho raha hai ya nahi
- website crash toh nahi ho rahi

Agar koi problem mile:
```text
Usko fix kiya jata hai
```

---

# Objectives of System Testing

1. Errors detect karna
2. Correct output verify karna
3. Performance improve karna
4. Security check karna
5. Reliability ensure karna

---

# Importance of System Testing

Agar testing na ho:
- software crash ho sakta hai
- wrong results aa sakte hain
- users system reject kar sakte hain

Good testing:
```text
High quality software provide karti hai
```

---

# Types of System Testing

1. Unit Testing
2. Integration Testing
3. System Testing
4. Acceptance Testing

---

# 1. Unit Testing

Unit Testing mein:
```text
Program ke small modules ko individually test kiya jata hai
```

---

# Example

Calculator app mein:
- addition function
- subtraction function

alag-alag test kiye jate hain.

---

# Advantages

1. Early bug detection
2. Easy debugging
3. Better code quality

---

# 2. Integration Testing

Integration Testing mein:
```text
Different modules ko combine karke test kiya jata hai
```

---

# Example

Online Shopping System mein:
- login module
- payment module
- order module

ek saath properly work kar rahe hain ya nahi check hota hai.

---

# 3. System Testing

System Testing:
```text
Complete software system ko test karta hai
```

---

# Example

Banking software ka:
- login
- money transfer
- balance checking

sab features test kiye jate hain.

---

# 4. Acceptance Testing

Acceptance Testing:
```text
Users ya clients ke dwara ki jati hai
```

Ye check karta hai:
```text
System user requirements fulfill karta hai ya nahi
```

---

# Example

School Management Software principal aur staff use karke verify karte hain.

# Introduction

