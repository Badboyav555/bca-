# Introduction to Feasibility Study

Before developing any software or system:
```text
It is important to check whether the project is possible and useful or not
```

This checking process is called:
```text
Feasibility Study
```

Feasibility study helps organizations decide:
- whether the project should continue
- whether the project is practical
- whether the project will give profit or benefit

---

# Real Life Example

Suppose a school wants:
```text
Online Attendance Management System
```

Before developing it, management checks:
- computers available?
- internet available?
- budget available?
- teachers can use system?

This complete checking process is:
```text
Feasibility Study
```

---

# Definition of Feasibility Study

Feasibility Study is:
```text
A process of evaluating a proposed system to determine whether it is practical, economical and technically possible
```

---

# Objectives of Feasibility Study

1. Check project possibility
2. Reduce risk of failure
3. Save time and money
4. Identify problems early
5. Help management in decision making

---

# Importance of Feasibility Study

Without feasibility study:
- project may fail
- budget may increase
- system may not work properly
- users may reject system

Feasibility study ensures:
```text
Better planning and successful system development
```

---

# Types of Feasibility

Main types:
1. Technical Feasibility
2. Operational Feasibility
3. Economic Feasibility

---

# 1. Technical Feasibility

# Introduction

Technical feasibility checks:
```text
Whether required technology, hardware and software are available or not
```

It determines:
- technical resources available?
- staff technically skilled?
- system technically possible?

---

# Example

Suppose a hospital wants:
```text
AI-based patient monitoring system
```

But:
- computers are outdated
- internet is slow
- staff lacks technical knowledge

Then project may not be:
```text
Technically feasible
```

---

# Questions Asked in Technical Feasibility

1. Is required hardware available?
2. Is required software available?
3. Is technology reliable?
4. Can existing staff handle system?
5. Is technical support available?

---

# Advantages of Technical Feasibility

1. Reduces technical failure
2. Identifies technology problems early
3. Improves system performance

---

# 2. Operational Feasibility

# Introduction

Operational feasibility checks:
```text
Whether users can operate and accept the new system
```

It studies:
- user acceptance
- employee training
- system usability

---

# Real Life Example

Suppose a company installs:
```text
Advanced payroll software
```

But employees:
- do not know computer operation

Then operational problems may occur.

---

# Questions Asked in Operational Feasibility

1. Will users accept system?
2. Is training required?
3. Will system improve work?
4. Is system easy to use?

---

# Importance of Operational Feasibility

Even if technology is excellent:
```text
Project may fail if users reject system
```

---

# Advantages

1. Improves user satisfaction
2. Reduces resistance from employees
3. Ensures smooth working

---

# 3. Economic Feasibility

# Introduction

Economic feasibility checks:
```text
Whether project is financially beneficial or not
```

It compares:
- project cost
- expected benefits

---

# Real Life Example

Suppose:
```text
Project Cost = ₹10 lakh
```

Expected benefit:
```text
₹50 lakh
```

Then project is economically feasible.

But if:
```text
Benefit = ₹1 lakh only
```

Then project may not be useful.

---

# Costs Included in Economic Feasibility

1. Hardware cost
2. Software cost
3. Employee salary
4. Training cost
5. Maintenance cost

---

# Benefits Included

1. Increased profit
2. Faster work
3. Reduced errors
4. Better customer service

---

# Advantages of Economic Feasibility

1. Controls unnecessary expenses
2. Improves investment decisions
3. Ensures project profitability

---

# System Performance Constraints

# Introduction

Constraints are:
```text
Limitations or restrictions affecting system performance
```

These limitations must be identified before development.

---

# Types of Performance Constraints

1. Hardware Constraints
2. Software Constraints
3. Time Constraints
4. Budget Constraints
5. Human Resource Constraints

---

# 1. Hardware Constraints

Limited:
- RAM
- processor speed
- storage

may affect system performance.

---

# Example

Old computers may not support:
```text
Heavy software applications
```

---

# 2. Software Constraints

Sometimes:
- software compatibility issues
- outdated operating systems

create problems.

---

# 3. Time Constraints

Projects must complete within:
```text
Fixed deadlines
```

---

# Example

College admission system must be ready:
```text
Before admission starts
```

---

# 4. Budget Constraints

Limited budget may reduce:
- software quality
- hardware quality
- staff hiring

---

# 5. Human Resource Constraints

Lack of skilled employees may affect:
```text
System development and maintenance
```

---

# COST BENEFIT ANALYSIS

# Introduction

Cost Benefit Analysis means:
```text
Comparing project costs with expected benefits
```

It helps determine:
```text
Whether project investment is worth it or not
```

---

# Real Life Example

Suppose a company spends:
```text
₹5 lakh
```

to develop billing software.

Benefits:
- faster billing
- reduced staff work
- improved customer service

If benefits are greater than cost:
```text
Project is beneficial
```

---

# Objectives of Cost Benefit Analysis

1. Identify project value
2. Compare expenses and profits
3. Improve management decisions
4. Reduce financial risk

---

# Difference Between Feasibility Study and Cost Benefit Analysis

| Feasibility Study | Cost Benefit Analysis |
|---|---|
| Checks project possibility | Compares cost and benefits |
| Includes technical factors | Focus on financial comparison |
| Broader concept | Part of feasibility study |

---

# Important Points to Remember

- Feasibility study checks project possibility.
- Technical feasibility checks technology availability.
- Operational feasibility checks user acceptance.
- Economic feasibility checks financial benefits.
- Constraints are system limitations.
- Objectives define system goals.
- Feasibility report explains project feasibility.
- Cost benefit analysis compares expenses and benefits.
- System proposal describes complete proposed system.
